## 1. 产品概述

冷链应急补冷 App 是面向冷藏车司机的移动端应急处置工具。当运输途中车厢温度超限时，司机可通过该应用按步骤完成异常上报、补冷方案选择与处置回执，减少电话沟通、标准化处置流程，保障冷链运输安全。

- 核心目标：让司机少打电话、按步骤处置，将异常响应时间从平均 25 分钟压缩至 10 分钟以内
- 目标用户：冷链物流企业签约冷藏车司机（30-55 岁，操作习惯偏向大按钮、简步骤）

## 2. 核心功能

### 2.1 用户角色

| 角色 | 注册方式 | 核心权限 |
|------|----------|----------|
| 司机 | 企业调度系统分配账号 | 异常上报、补冷导航、处置回执提交 |
| 调度员 | 企业调度系统分配账号 | 查看回执、判断处置方案（本版本仅数据展示，不做独立页面） |

### 2.2 功能模块

1. **异常上报页**：温度超限提醒、车牌选择、货品类型、当前箱温、预计剩余里程、照片拍摄
2. **补冷导航页**：附近补冷方案列表（干冰点/制冷剂补给点/合作冷库/安全停车区）、方案详情与选择、导航入口
3. **处置回执页**：任务清单打勾（停车检查→联系站点→补冷完成→复测温度）、补冷数量录入、复测温度录入、照片上传、提交回执

### 2.3 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|----------|----------|----------|
| 异常上报 | 温度超限提醒横幅 | 红色闪烁横幅显示"车厢温度超限"，持续提醒司机 |
| 异常上报 | 车牌选择 | 下拉选择或输入当前驾驶车辆车牌号 |
| 异常上报 | 货品类型选择 | 预设选项：冷冻肉类、冷藏蔬果、乳制品、医药制品、速冻食品、其他 |
| 异常上报 | 当前箱温输入 | 数字输入框，带温度范围提示（-25℃ ~ 15℃） |
| 异常上报 | 预计剩余里程 | 数字输入框，单位 km |
| 异常上报 | 照片拍摄 | 拍摄温控屏或封签照片，最多3张，支持预览与删除 |
| 异常上报 | 提交按钮 | 大按钮，提交后跳转至补冷导航页 |
| 补冷导航 | 方案类型筛选 | 四种类型标签切换：干冰点、制冷剂补给点、合作冷库、安全停车区 |
| 补冷导航 | 方案列表卡片 | 每个方案显示：名称、距离/绕行时间、营业状态、可补冷容量、联系人电话 |
| 补冷导航 | 方案详情弹窗 | 点击卡片展开详情：地址、营业时间、可补冷类型、容量、一键拨号、导航按钮 |
| 补冷导航 | 确认选择按钮 | 选择方案后确认，跳转至处置回执页 |
| 处置回执 | 任务清单 | 四步清单：①停车检查 ②联系站点 ③补冷完成 ④复测温度，逐步打勾解锁下一步 |
| 处置回执 | 补冷数量录入 | 数字输入，选择单位（kg/瓶/罐） |
| 处置回执 | 复测温度录入 | 数字输入框，系统判断是否恢复安全范围 |
| 处置回执 | 照片上传 | 拍摄补冷后车厢/温控屏照片，最多3张 |
| 处置回执 | 提交回执按钮 | 提交后显示调度员判断结果：继续运输/换车/转入临时冷库 |

## 3. 核心流程

司机收到温度超限推送 → 打开App查看异常详情 → 填写车牌、货品类型、当前箱温、剩余里程 → 拍摄温控屏/封签照片 → 提交异常上报 → 系统推荐附近补冷方案 → 司机按类型筛选查看 → 选择合适方案确认 → 进入处置回执页 → 按步骤打勾完成任务清单 → 录入补冷数量和复测温度 → 上传照片 → 提交回执 → 等待调度员判断

## 4. 用户界面设计

### 4.1 设计风格

- **主色调**：深蓝 #0F2B46（冷链专业感）+ 冰蓝 #00B4D8（科技冷感）+ 警示橙 #FF6B35（异常告警）
- **辅助色**：冷白 #F0F7FF（背景）、深灰 #1B2A3D（文字）、成功绿 #00C853
- **按钮风格**：大圆角（12px），核心操作按钮带微光效果，高对比度
- **字体**：标题使用 DIN Alternate（数字感强），正文使用系统默认
- **布局风格**：移动端单列布局，卡片式分区，底部固定操作栏
- **图标风格**：线性图标（Lucide），2px 描边，冰蓝色主题
- **整体氛围**：工业仪表盘风格，暗色主题，温度数值突出显示，紧急感与专业感并存

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI元素 |
|----------|----------|--------|
| 异常上报 | 温度超限横幅 | 红色脉冲动画背景，白色大字"温度超限警告"，温度数值闪烁 |
| 异常上报 | 信息录入区 | 暗色卡片，白色文字，冰蓝色输入框边框，大触摸区域 |
| 异常上报 | 照片拍摄区 | 虚线框+相机图标，已拍照片缩略图预览，右上角删除按钮 |
| 异常上报 | 提交按钮 | 底部固定，橙色大按钮，按下态变深 |
| 补冷导航 | 类型筛选标签 | 横向滚动标签栏，选中态冰蓝底白字，未选透明底灰字 |
| 补冷导航 | 方案卡片 | 暗色卡片，左上角类型图标，右侧营业状态灯（绿/红），底部联系人一键拨号 |
| 补冷导航 | 确认按钮 | 底部固定，选中方案后激活为橙色，未选中时灰色不可点击 |
| 处置回执 | 任务清单 | 竖向步骤条，已完成步骤绿色勾+灰字，当前步骤冰蓝高亮，未完成步骤灰色 |
| 处置回执 | 数据录入区 | 与异常上报风格一致 |
| 处置回执 | 提交回执按钮 | 底部固定，全部步骤完成后激活为绿色 |
| 处置回执 | 结果弹窗 | 全屏模态，三种结果卡片：继续运输（绿）、换车（橙）、转入冷库（蓝） |

### 4.3 响应式设计

- 移动端优先（375px - 428px 宽度为核心适配范围）
- 触摸优化：所有可点击区域最小 44px × 44px
- 单手操作：核心按钮置于屏幕下半区
- 横屏自适应：信息区与操作区并排布局

### 4.4 动效设计

- 页面切换：右滑进入/左滑退出
- 温度超限横幅：红色脉冲动画（1.5s 循环）
- 步骤打勾：弹性缩放 + 绿色扩散
- 卡片点击：轻微下沉反馈
- 提交成功：对勾弹出 + 渐变收起
